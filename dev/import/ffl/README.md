# dev/import/ffl

Tooling and source data for importing FFL season data from Tapatalk forum posts into the dev database.

## Scripts

**`parse_forum.py`** — Parses raw Tapatalk forum posts into structured CSVs.
- Input: `<year>_<round>.txt` (copy-pasted forum thread)
- Output: `<year>_<round>_teams.csv` (one row per player) + `<year>_<round>_scores.csv` (one row per team)
- Handles all 4 team posting formats: Ruiboys (dash-separated), Slashers (bracketed), Cheetahs (parenthesised with raw stats), THC (equals/dash-separated)

**`resolve_squads.py`** — Matches squad player names to `afl.player` records and generates seed SQL.
- Mode 1: `python3 resolve_squads.py` — reads `2026_squads.txt`, fuzzy-matches to `afl.player`, writes `squads_review.csv`
- Mode 2: `python3 resolve_squads.py --sql squads_review.csv` — generates `04_ffl_players.sql` (all IDs resolved by name, no hardcoded integers)
- Review the CSV and correct any fuzzy/no-match rows before generating SQL

**`import_round_teams.py`** — Converts parsed round CSVs into `ffl.player_match` + `ffl.club_match` score SQL.
- `python3 import_round_teams.py` — generates SQL to stdout (pipe to `06_ffl_round_data.sql`)
- `python3 import_round_teams.py --apply` — applies directly to dev DB
- All IDs resolved by name via subqueries; safe to re-run after a `dev-seed`

## Source data

| File | Contents |
|------|----------|
| `2026_squads.txt` | Initial 2026 squad lists for all 4 clubs (format: `Player Name price ClubCode`) |
| `squads_review.csv` | Resolved squad → `afl.player` mapping; source of truth for 04_ffl_players.sql |
| `2026_N.txt` | Raw copy-pasted Tapatalk forum thread for round N |
| `2026_N_teams.csv` | Parsed player selections for round N |
| `2026_N_scores.csv` | Parsed team scores for round N |

## Seed files generated

These files live in `dev/postgres/seed/` and are generated by the scripts above — do not edit manually:

| File | Contents |
|------|----------|
| `04_ffl_players.sql` | `ffl.player` + `ffl.player_season` for all 4 squads (120 players) |
| `05_ffl_trades.sql` | Mid-season trade additions (manually authored) |
| `06_ffl_round_data.sql` | `ffl.player_match` + `ffl.club_match` scores for R1–R5 |

## Workflow for a new round

1. Copy-paste the forum thread into `2026_<N>.txt`
2. `python3 parse_forum.py 2026_<N>.txt` → review the CSV output
3. `python3 import_round_teams.py --apply` to load into dev DB
4. Regenerate `06_ffl_round_data.sql`: `python3 import_round_teams.py > ../../postgres/seed/06_ffl_round_data.sql`
